# Vocabulary for Generative AI Applications

## 1. Generative AI
Models that create new content such as text, images, audio, or code.

## 2. Large Language Model
Neural network trained on massive datasets to understand and generate natural language.

## 3. Foundation Model
Large pre-trained model adaptable to downstream tasks.

## 4. Embeddings
Vector representation of meaning for text or images.

## 5. Tokenization
Splitting text into smaller units (tokens).

## 6. Context Window
Max number of tokens a model can process at once.

## 7. Prompt Engineering
Designing prompts to control model behavior.

## 8. Chain of Thought
Prompting method that forces the model to show reasoning steps.

## 9. Few-Shot Learning
Learning with a small number of examples.

## 10. Zero-Shot Learning
Performing a task without examples.

## 11. Hallucination
Incorrect or fabricated model output.

## 12. RAG (Retrieval-Augmented Generation)
Method combining retrieval + generation to reduce hallucinations.

## 13. Semantic Search
Search based on meaning rather than keywords.

## 14. Vector Database
Database for storing and searching embeddings.

## 15. Diffusion Model
Model used to generate images.

## 16. Fine-Tuning
Additional training on domain-specific data.

## 17. Alignment
Ensuring model behavior follows human values.

## 18. Latency
Time required to produce model output.

## 19. Inference
Running a trained model to generate results.

## 20. GPU Acceleration
Using GPUs to speed up AI computations.

## 21. Multimodal Model
Model that processes multiple types of data (text, image, audio).

## 22. Model Drift
Performance degradation over time.

## 23. Knowledge Base
Source of external facts for RAG systems.

## 24. Personalization
Adjusting output based on user context.

## 25. Summarization
Creating a shorter version of text.

## 26. Classification
Assigning labels to data.

## 27. Parsing
Extracting structure from text.

## 28. Grounding
Connecting model output to real data.

## 29. API Endpoint
URL interface to call an AI model.

## 30. Safety Guardrails
Filters preventing unsafe model behavior.

## 31. Function Calling
LLM triggering external functions via JSON.

## 32. Orchestration
Automation of multi-step AI workflows.

## 33. Optimization
Improving model or pipeline performance.

## 34. Evaluate
Assessing the quality of models.

## 35. Monitor
Tracking system metrics and drift.

## 36. Generate
Creating new content using a model.

## 37. Retrieve
Selecting relevant information from a dataset.

## 38. Transform
Converting data formats.

## 39. Validate
Checking correctness of outputs.
